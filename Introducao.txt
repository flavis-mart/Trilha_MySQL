1. Explique o que é MySQL e cite ao menos três de suas vantagens.
- MySQL se trata de um SGDB (sigla para Software de Gerenciamento de Banco de Dados) relacional.
- Dentre suas principais vantagens estão: Alta performance, significando que possui capacidade de
- lidar com grandes quantidades de operações e dados, mantendo a fluidez e velocidade adequadas; 
- Integração com diversas linguagens, isso é uma vantagem por ser um grande facilitador, já que com
- apenas um SGDB é possível trabalhar com tecnologias diferentes, servindo para propósitos diferentes; 
- Grande comunidade e vasta documentação; faz com que aprender e lidar com erros seja mais fácil, pois
- a comunidade e a documentação dificilmente não terá uma resposta para suas dúvidas e problemas.

2. Qual a diferença entre um banco de dados relacional e um banco de dados não relacional? Dê exemplos práticos de uso.
- Entre as principais diferenças entre esses dois bancos de dados está nas estruturas, onde:
- Banco de dados relacional -> utiliza tabelas com esquemas bem definidos e relações explícitas;
- Banco de dados não relacional -> utiliza uma estrutura mais flexível, já que foi criado para lidar com grandes volumes de dados;
- Exemplo prático: 
- Um banco de dados relacional organizaria os dados em tabelas, por exemplo os dados de uma sorveteria:
- Haveria uma tabela com todos os dados dos produtos, clientes, vendas, estoques.
- Enquanto o bando de dados não relacional trabalharia de forma mais flexível com documentos parecidos com um JSON,
- agrupando as informações:
- Imagine que exista um documento com a coleção de dados dos sorvetes -> {sabor: flocos, litros: 2, marca: kibom}

3. Cite e explique os principais componentes da arquitetura do MySQL.
- Tabelas: estruturas que armazenam os conjuntos de registros ou dados;
- Colunas: significam os campos de informação, por exemplo: coluna nome da tabela clientes;
- Linhas: juntando todos os dados das colunas temos um registro, seguindo o exemplo acima, todos os dados de um cliente;
- Chaves primária e estrangeira: a primária é única e por isso identifica uma tabela, enquanto a estrangeira cria um relacionamento,
- uma ponte, conexão entre uma tabela e outra;
- Índices: servem como uma espécie de atalho para localizar os dados.

4. Qual a função do motor de armazenamento do MySQL? Explique a diferença entre InnoDB e MyISAM.
- O motor de armazenamento é aquele que trata: como os dados e os índices são armazenados no disco, o controle de concorrência,
- o suporte das transações e recuperação dos dados após falhas no servidor.
- Diferenças entre MyISAM e InnoDB:
- basicamente o InnoDB hoje ganha do MyISAM em questão de vantagens, ele: suporta transações (ACID) enquanto o MyISAM não, seu nível
- de bloqueio permite que mais de uma pessoa altere dados na mesma tabela simultâneamente (desde que não na mesma linha), tem uma recuperação
- de falhas melhor e mais confiável, e a lista continua.

8. Como criar uma conexão no MySQL Workbench? Liste o passo a passo.
- Na tela inicial (Home), procure pela seção central chamada "MySQL Connections";
- Clique no ícone de "+" (sinal de adição) pequeno que fica ao lado do título "MySQL Connections";
- Preencha os dados na janela "Setup New Connection";
- No canto inferior direito dessa mesma janela, clique no botão "Test Connection";
- Se tudo estiver certo, aparecerá uma caixa de diálogo dizendo "Successfully made the MySQL connection";
- Por fim: salve a conexão. Após o teste bem-sucedido, feche a mensagem de sucesso e clique no botão "OK" na janela principal de configuração.

11. Dê um exemplo prático de conexão ao MySQL utilizando o DBeaver. Quais são os parâmetros necessários para essa conexão?
- Você primeiro clica no ícone de "Nova Conexão" no canto superior esquerdo e seleciona o MySQL;
- Na aba "Main" (Principal) que se abrirá, você precisará preencher os seguintes parâmetros necessários:
- Host (Servidor): O endereço onde o banco de dados está hospedado. Se for na sua máquina, use localhost ou 127.0.0.1.
- Port (Porta): A porta de comunicação. O padrão do MySQL é 3306.
- Database (Banco de Dados): O nome do banco específico que você quer abrir.
- Username (Usuário): Seu usuário de acesso (ex: root).
- Password (Senha): A senha correspondente ao usuário.


12. Durante a conexão via terminal, o comando mysql -u root -p é utilizado. Explique o que cada parte desse comando significa.

13. Cite as principais diferenças entre o MySQL Workbench, o DBeaver e o phpMyAdmin. Qual deles é mais indicado para o uso web?
- Suas principais diferenças são 
-> MySQL Workbench: Seu grande diferencial é ser construído especificamente para o MySQL;
-> DBeaver: é uma ferramenta universal. Seu foco é a produtividade do desenvolvedor;
-> phpMyAdmin: É uma aplicação web construída em PHP. Ele não é instalado no seu 
- computador pessoal como um programa, mas sim no servidor web, e você o acessa pelo navegador.
- Conclusão: o mais indicado para o uso web é justamente aquele que se destaca por essa característica: o phpMyAdmin.

15. Explique o conceito de buffer de memória no MySQL e sua importância para a performance do banco de dados.
- É um grande espaço reservado na memória RAM do servidor que funciona como um cache. O buffer de memória resolve
- a operação mais lenta do banco: buscar dados no disco rígido. Ele guarda os dados mais procurados na RAM. Quando
- a aplicação pede uma informação que já está no buffer, o MySQL a entrega muito mais rápido, garantindo a alta velocidade das consultas.
